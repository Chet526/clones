# Security Policy

GeoBrief LE takes security and evidentiary integrity seriously.

## Reporting a Vulnerability

Report potential vulnerabilities to:
- security@geobriefle.com

Please include:
- Affected component and version
- Reproduction steps
- Impact assessment
- Any proof-of-concept details

Do not publicly disclose vulnerabilities until we confirm remediation guidance.

## Response Targets

- Initial acknowledgement: within 2 business days
- Triage update: within 5 business days
- Remediation timeline: provided after severity assessment

## Scope

This policy covers:
- The GeoBrief LE repository and released builds
- Storefront/serverless functions in site/functions
- Security-sensitive configuration guidance in docs and README

Out-of-scope:
- Third-party providers (Stripe, Supabase, Netlify) platform-level issues
- Social-engineering tests against staff
